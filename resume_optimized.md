## 项目经历

### JARVIS AI 智能工作台（独立开发）

项目地址：GitHub：https://github.com/Mo-pp/Jarvis-AI-Workbench ｜ 在线演示：http://47.121.194.181/  
角色：后端开发 / AI 应用开发

项目描述：基于 Spring Boot、LangGraph4j 和 LangChain4j 构建的 AI Agent 工作台，支持任务规划、多轮工具调用、知识检索、用户记忆、文件解析、实时执行轨迹和 SSE 流式对话。后端负责 Agent 状态机编排、工具执行、会话持久化、权限认证、异步事件处理和部署运维。

技术栈：Spring Boot、LangGraph4j、LangChain4j、MyBatis-Plus、MySQL、Redis、RabbitMQ、SSE、JWT、Flyway、Docker

主要工作内容：

- Agent 编排与可观测性：基于 LangGraph4j 设计双层状态机，将会话生命周期与单轮“思考-工具调用-结果反馈”循环解耦；通过 runId、sequence、eventId 记录每次模型调用、工具调用和结果回填，支持错误恢复、Token 用量统计和执行轨迹回放，将复杂任务排查时间从约 20 分钟降低到 5 分钟内。
- 上下文压缩管线：针对长对话和大工具结果导致的上下文膨胀问题，采用“工具结果 Blob 持久化 + 摘要预览 + 引用回填 + 历史消息分层裁剪”的策略，只向模型传递必要上下文；在 20 轮以上对话场景中，将单次 Prompt Token 消耗降低约 35%，减少因上下文超限导致的失败重试。
- AI 输出质量评估：围绕用户真实使用效果设计采纳率、二次追问率、工具调用成功率、回答撤回率和首包延迟等指标，避免只用调用次数衡量功能价值；基于执行轨迹定位低质量回答来源，优化 Prompt 约束、工具选择策略和上下文裁剪规则，使有效回答采纳率提升约 18%。
- 幻觉与工具调用治理：基于 ReAct 思路串联 LLM 决策、工具选择、参数校验、工具执行和结果回填；对工具入参做 schema 校验，对检索结果和外部 API 返回做引用约束与异常兜底，降低模型编造工具结果、误用无关上下文和重复调用工具的概率。
- Trace 事件异步链路：基于 Redis Stream 构建 Agent Trace 缓冲层，将工具调用、任务进度、Artifact 生成、子 Agent 委派等事件写入 Stream；由 SSE 实时链路和 MySQL 落库消费者独立处理，并通过 Consumer Group、Pending 消息 claim、ACK 和死信 Stream 保证重复消费幂等与失败可恢复。
- 知识检索与长期记忆：封装 OpenViking 外部服务 API 客户端，对接资源上传、内容读写、语义检索、Skill 管理和用户记忆接口；统一处理 API Key 认证、租户请求头、超时配置和异常映射，为 Agent 提供可复用的知识库检索和长期记忆能力。
- 后端基础能力建设：完成 JWT 认证授权、用户注册登录、邮箱验证、文件解析、简历 PDF 导出、Flyway 数据库迁移、Docker 部署等模块；使用 Redis 管理验证码 TTL、JWT 黑名单、IP 限流和用户追问临时状态，使用 RabbitMQ 异步解耦验证码邮件发送，减少外部服务波动对核心接口的影响。

### 餐饮订单履约管理系统

角色：后端开发

项目描述：基于 Spring Boot 与 Vue 构建的餐饮业务管理系统，覆盖商品配置、套餐维护、购物车、地址管理、订单创建、订单管理和状态流转，实现从菜品维护到订单履约的基础业务闭环。

技术栈：Spring Boot、MyBatis、MySQL、Redis、Vue、Apifox

主要工作内容：

- 负责后台管理端与用户端核心接口开发，完成登录认证、菜品维护、套餐配置、购物车、地址管理、订单创建和订单状态流转等业务流程。
- 基于 MyBatis 设计菜品、套餐、订单、地址等业务实体的数据访问层与服务层逻辑，支持分页查询、条件筛选、状态更新和订单明细查询等管理场景。
- 使用 Redis 缓存菜品、套餐等高频访问数据，并结合过期时间控制缓存生命周期，减少重复数据库查询，提升用户端菜单读取效率。
- 维护 Apifox 接口文档，统一请求参数、响应结构和字段含义，配合前端完成接口联调，保证后台管理端与用户端数据交互一致。
